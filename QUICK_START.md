# 🚀 Design Bridge 빠른 시작 가이드

## ⚡ 3단계로 시작하기

### 1️⃣ 의존성 설치



## 📚 현재 버전 특징

### ✅ 로컬 데이터 기반
- **API 키 불필요** - 별도 설정 없이 즉시 사용
- **30개 핵심 용어** - 주요 그래픽 디자인 용어 탑재
- **사전 형태 제공** - 모든 용어를 한눈에 확인
- **빠른 응답** - 네트워크 지연 없음

---

## 💡 사용 방법

### 방법 1: 용어 사전에서 클릭
1. 화면에 표시된 "사용 가능한 용어 (30개)" 섹션 확인
2. 원하는 용어 클릭
3. 자동으로 번역 결과 표시

### 방법 2: 직접 검색
1. 검색창에 용어 입력
2. "번역하기" 버튼 클릭 또는 Enter
3. 번역 결과 확인

### 방법 3: 번역 방향 전환
1. 상단 "학술 → 실무" 버튼 클릭
2. 방향 전환 (실무 → 학술)
3. 반대 방향으로 검색

## 🎯 기능 소개

### 1. 양방향 번역
- **학술 → 실무**: 교재 용어를 실무 언어로
- **실무 → 학술**: 현장 용어의 이론적 배경

### 2. 실무 예시 제공
각 용어마다 3가지 상황별 예시:
- 디자인 회의
- 클라이언트 피드백
- 업무 지시

### 3. 검색 히스토리
- 최근 검색한 용어 자동 저장
- 빠른 재검색 가능
- 로컬 스토리지에 저장

### 4. 반응형 디자인
- 모바일 📱
- 태블릿 💻
- 데스크톱 🖥️

## 🎨 사용 예시

### 예제 1: "시각적 위계" 검색

1. 용어 사전에서 "시각적 위계" 클릭
2. 결과 확인:
   - **학술 용어**: 시각적 위계 (Visual Hierarchy)
   - **실무 용어**: 정보 우선순위, 시선 흐름, 강조 포인트
   - **설명**: 어떤 정보를 먼저 보여줄지 결정하는 것
   - **실무 예시**: "제목을 크게 해서 시선이 먼저 가게 해주세요"

### 예제 2: "게슈탈트 법칙" 검색

1. 검색창에 "게슈탈트" 입력
2. 번역하기 클릭
3. 결과 확인:
   - **실무 용어**: 그룹감, 관련 요소 묶기, 레이아웃 정리
   - **실무 예시**: "관련된 정보들은 가까이 배치해서 그룹감을 줍시다"

## ❓ 자주 묻는 질문

### Q: API 키가 필요한가요?
A: **아니요!** 현재 버전은 로컬 데이터 기반이라 API 키가 필요 없습니다.

### Q: 용어가 30개밖에 없나요?
A: 현재 버전은 MVP로 30개 핵심 용어를 포함합니다. 향후 AI API 연동 시 무제한 용어 지원 예정입니다.

### Q: 새로운 용어를 추가할 수 있나요?
A: 네! `app/data/terms.ts` 파일에 용어를 추가하면 됩니다. 기존 형식을 참고하세요.

### Q: 검색 히스토리는 어디에 저장되나요?
A: 브라우저의 로컬 스토리지에 저장됩니다. 브라우저를 닫아도 유지됩니다.

### Q: 모바일에서도 사용할 수 있나요?
A: 네! 반응형 디자인으로 모든 기기에서 최적화되어 있습니다.

## 🐛 문제 해결

### 서버가 시작되지 않아요
```bash
# node_modules 삭제 후 재설치
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 포트가 이미 사용 중이에요
```bash
# 다른 포트로 실행
PORT=3001 npm run dev
```

### 용어 검색이 안 돼요
- 용어 사전에서 정확한 용어를 클릭하세요
- 부분 매칭도 지원하지만 정확한 용어를 사용하는 것이 좋습니다

```typescript
  contexts: [
    { situation: "상황1", example: "예시1" },
    { situation: "상황2", example: "예시2" },
    { situation: "상황3", example: "예시3" }
  ]
}
```
