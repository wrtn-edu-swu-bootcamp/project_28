# Design Bridge 테스트 가이드 🧪

## 테스트 시나리오

### 1. 기본 기능 테스트

#### 1.1 학술 → 실무 번역

**테스트 케이스 1: 기본 용어**
- 입력: "시각적 위계"
- 기대 결과:
  - 실무 용어 표시 (예: "정보 우선순위", "시선 흐름")
  - 설명 문구
  - 3가지 실무 상황 예시

**테스트 케이스 2: 복잡한 용어**
- 입력: "게슈탈트의 근접성 원리"
- 기대 결과: 학술 개념을 실무 언어로 쉽게 설명

**테스트 케이스 3: 색채 관련**
- 입력: "색채 대비"
- 기대 결과: 실무에서 사용하는 컬러 관련 표현

#### 1.2 실무 → 학술 번역

**테스트 케이스 1: 실무 표현**
- 토글 전환 후 입력: "여백 넉넉하게"
- 기대 결과: "네거티브 스페이스", "여백(White Space)" 등 학술 용어

**테스트 케이스 2: 업계 용어**
- 입력: "레퍼런스"
- 기대 결과: 학술적 의미 설명

**테스트 케이스 3: 작업 지시**
- 입력: "누끼따기"
- 기대 결과: 기술적 배경과 이론적 근거

### 3. 에러 처리 테스트

#### 3.1 입력 검증

**테스트 케이스 1: 빈 입력**
- 입력: (빈 칸)
- 기대 결과: "검색어를 입력해주세요" 오류 메시지

**테스트 케이스 2: 너무 긴 입력**
- 입력: 100자 이상의 텍스트
- 기대 결과: "검색어가 너무 깁니다" 오류 메시지

#### 3.2 API 오류

**테스트 케이스 1: 잘못된 API 키**
- `.env.local`에 잘못된 키 입력
- 기대 결과: "API 인증 오류" 메시지

**테스트 케이스 2: 네트워크 오류**
- 인터넷 연결 끊기
- 기대 결과: 적절한 에러 메시지

#### 3.3 Rate Limiting

**테스트 케이스: 연속 요청**
- 1분 내 10회 이상 검색
- 기대 결과: "너무 많은 요청입니다" 메시지

## 자동화 테스트 예제

### Jest 단위 테스트 (향후 추가 가능)

\`\`\`typescript
// __tests__/api/translate.test.ts
describe('Translation API', () => {
  it('should return valid translation', async () => {
    const response = await fetch('/api/translate', {
      method: 'POST',
      body: JSON.stringify({
        term: '시각적 위계',
        direction: 'academic-to-practical'
      })
    });
    
    const data = await response.json();
    expect(data.academic_term).toBeDefined();
    expect(data.practical_terms).toBeInstanceOf(Array);
  });
});
