# Design Bridge - 프로젝트 요약

## 📊 프로젝트 개요

**프로젝트명**: Design Bridge  
**목적**: 그래픽 디자인 전공 학생들을 위한 학술-실무 언어 번역 서비스  
**타겟**: 디자인 전공 대학생, 신입 디자이너  
**개발 기간**: 2일 (MVP)  
**현재 상태**: ✅ 완료 (로컬 데이터 기반)

---

## 🎯 핵심 가치

### 문제 정의
- 학교에서 배운 이론 용어와 실무 언어의 괴리
- 신입 디자이너의 커뮤니케이션 어려움
- 교수와 선배 디자이너의 언어 차이

### 솔루션
- **양방향 번역**: 학술 ↔ 실무 용어 상호 변환
- **실무 맥락**: 실제 상황별 사용 예시 제공
- **즉시 사용**: API 키 없이 바로 사용 가능

---

## 🛠️ 기술 스택

### Frontend & Backend
- **프레임워크**: Next.js 14 (App Router)
- **언어**: TypeScript
- **스타일링**: Tailwind CSS
- **데이터**: 로컬 TypeScript 모듈

### 배포
- **플랫폼**: Vercel (추천)
- **도메인**: vercel.app 또는 커스텀 도메인

---

## 📦 현재 버전 특징

### ✅ 로컬 데이터 기반 (v1.0)
- **30개 핵심 용어** 탑재
- **API 키 불필요**
- **빠른 응답 속도**
- **오프라인 작동 가능**

### 장점
1. ✅ **즉시 사용 가능** - 복잡한 설정 불필요
2. ✅ **안정성** - API 의존성 없음
3. ✅ **빠른 속도** - 네트워크 지연 없음
4. ✅ **비용 절감** - API 비용 제로

### 제약사항
1. ❌ 30개 용어로 제한
2. ❌ 동적 용어 생성 불가
3. ❌ 사용자 맞춤 예시 불가

## 🎨 주요 기능

### 1. 용어 사전 📚
- 30개 용어를 태그 형태로 표시
- 클릭 한 번으로 바로 검색
- 접기/펼치기 기능
- 반응형 레이아웃

### 2. 양방향 번역 🔄
- **학술 → 실무**: 이론 용어를 현장 언어로
- **실무 → 학술**: 실무 용어의 이론적 배경
- 버튼 클릭으로 방향 전환

### 3. 실무 예시 💼
각 용어마다 3가지 상황:
- 디자인 회의
- 클라이언트 피드백
- 업무 지시

### 4. 검색 히스토리 🔍
- 최근 검색 자동 저장
- 로컬 스토리지 활용
- 빠른 재검색

### 5. 반응형 UI 📱
- 모바일 최적화
- 태블릿 지원
- 데스크톱 확장

## 🎯 차별화 포인트

### vs. 일반 번역기
- ❌ 일반 번역: 단순 언어 변환
- ✅ Design Bridge: **맥락과 예시 제공**

### vs. 용어 사전
- ❌ 일반 사전: 정의만 제공
- ✅ Design Bridge: **실무 상황 예시 포함**

### vs. 교재
- ❌ 교재: 이론 중심
- ✅ Design Bridge: **이론 + 실무 연결**
